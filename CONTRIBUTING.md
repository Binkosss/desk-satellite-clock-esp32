# Contributing

Thanks for contributing.

## Workflow
1. Fork the repository
2. Create a feature branch
3. Keep commits small and focused
4. Open a Pull Request with clear description

## Code Rules
- Keep code and comments in concise English
- Do not commit private credentials
- Store Wi-Fi secrets only in `include/secrets.h`
- Keep `src/` for active source code only

## Local Check
Before opening a PR:
- Build project with PlatformIO
- Confirm serial output is readable
- Confirm mode switching and LED behavior still works
